## 1 总览：完整数据流

每个实验都遵循同一个 5-stage 流水线：

```
┌─────────────────┐    ┌──────────────┐    ┌──────────────┐    ┌──────────────┐    ┌──────────────┐
│   ① 数据生成    │ → │ ② Corruption │ → │ ③ Imputation │ → │ ④ Forecasting│ → │   ⑤ 度量     │
│  (混沌集成器)   │    │  (v2 grid)   │    │ (lin/CSDI/   │    │ (Panda/Chro/ │    │ (VPT/vh/CI)  │
│                 │    │              │    │  SAITS/...)  │    │  DeepEDM/Enkf│    │              │
└─────────────────┘    └──────────────┘    └──────────────┘    └──────────────┘    └──────────────┘
   shape: (T, D)        shape: (T, D)+      shape: (T, D)       shape: (H, D)        scalar / dist
   T 长度轨迹           NaN at masked       no NaN              future trajectory     paired bootstrap
                        time-steps                                                    Wilson CI
```

**举例**（L63 SP65 baseline）：
```
① integrate_lorenz63(seed=0, T=512+128=640)         → traj: (640, 3)
② make_corrupted_observations(traj[:512], s=0.65, σ=0)  → observed: (512, 3) with NaN
③ impute(observed, kind="linear")                    → filled: (512, 3) no NaN
④ panda_forecast(filled, pred_len=128)               → forecast: (128, 3)
⑤ valid_prediction_time(traj[512:], forecast, ...)   → vpt: float (in Lyapunov units)
```

每个 cell（一种 imputer × forecaster × scenario × seed）就是一次完整流水线。多个 seed 的结果聚合后给出 mean / median / Pr(VPT > τ) / paired bootstrap CI。

## 3 Corruption 协议（v2）

### 3.1 corruption_grid_v2.json

**v2 grid** 是论文的锁定 corruption 协议，定义在
`experiments/week1/configs/corruption_grid_v2.json`。

包含 4 个 cell groups：
- `legacy_diagonal`：旧 S0–S6 protocol 的兼容入口
- `fine_s_line`：纯稀疏度扫描，σ=0
- `fine_sigma_line`：纯噪声扫描，s=0
- `summary_path_candidate`：headline cells（SP65, SP82 等）

每个 cell 字典：
```json
{"name": "SP65", "sparsity": 0.65, "noise_std_frac": 0.0, ...}
```

`grid_index` = cell 在 grid pool 中的位置（用于构造 corruption seed）。

### 3.2 Corruption 函数

`experiments/week1/corruption.py::make_corrupted_observations`

**输入**：clean `traj` shape (T, D)
**输出**：`obs_res` 带 `observed` (T, D) with NaN at masked, plus metadata

**关键参数**：
- `sparsity` (s)：要 mask 掉的时步比例
- `noise_std_frac` (σ)：相对 attractor_std 的噪声尺度
- `mask_regime`：`"iid_time"`（独立时步）或 `"patch"`（块状缺失）
- `attractor_std`：系统的 σ_attr，用于把 σ 还原为绝对噪声
- `seed`：corruption RNG seed（**与 trajectory seed 不同**）
- `patch_length`：块状缺失时块大小

**Corruption seed scheme**（论文锁定）：
```
seed_corr = 1000 × seed + 5000 + grid_index
```
`grid_index` 从 v2 grid JSON 读出。这保证不同 cell 的 mask 不一样，但 same seed 跨实验可复现。

### 3.3 Mask 行为

```python
# pseudo-code from corruption.py
keep_mask = (rng.random(T) > sparsity)   # 保留 (1−s) 比例的时步
noise = rng.normal(0, σ * attractor_std, (T, D))
observed = traj + noise
observed[~keep_mask] = NaN
```

`keep_frac` = (1 − sparsity) 实际值（≈），存到 metadata。

### 3.4 σ_override 与 anchor 精度（关键陷阱）

CSDI 推理时需要传入 `sigma_override = noise_std_frac × attractor_std`：
- 纯稀疏 cell（σ=0）→ `sigma_override = 0`，CSDI 在观测时步保持 anchor 精度 ~10⁻⁶
- 噪声 cell（σ>0）→ `sigma_override > 0`，CSDI 知道观测自身有噪声，不会过度信任

如果 `sigma_override` 误传或缺省，CSDI 会过 / 欠 noised — 是早期协议不一致的源头（在 `deliverable/CSDI_SANITY_FINDINGS.md` 中诚实记录）。

## 4 Imputation 模块

### 4.1 统一入口

`methods/dynamics_impute.py`：

```python
from methods.dynamics_impute import impute
filled = impute(observed, kind="linear")
filled = impute(observed, kind="ar_kalman", ar_order=5)
filled = impute(observed, kind="csdi", sigma_override=...)
```

支持的 `kind`：
- `linear`：scipy 1D 插值，per-channel 独立
- `ar_kalman`：AR(p) Kalman smoother，p 默认 5
- `csdi`：调用 `methods/csdi_impute_adapter.py`（详见 §4.2）

### 4.2 CSDI 推理

**全局状态**：CSDI checkpoint 与 attractor_std 通过 module-level setter 设置：
```python
from methods.csdi_impute_adapter import set_csdi_checkpoint, set_csdi_attractor_std
set_csdi_checkpoint("experiments/week2_modules/ckpts/dyn_csdi_full_v6_center_ep20.pt")
set_csdi_attractor_std(8.51)
```

之后 `impute(observed, kind="csdi", sigma_override=...)` 会调用 CSDI inference：
- 输入 `observed` shape (T, D)
- 内部 forward diffusion + reverse sampling
- 返回 filled shape (T, D)

**关键点**：每个 system 用对应的 CSDI ckpt（见 §7 训练）。

### 4.3 SAITS-pretrained 推理（chunked）

SAITS 训练时固定 `n_steps=128`（位置编码长度匹配）。测试 context 长度 512，所以推理时切 4 个非重叠 length-128 chunk，独立补全后拼接：

```python
# from experiments/week1/panda_altimputer_control.py:_saits_pretrained_impute
chunks = observed_2d.reshape(n_chunks=4, train_n_steps=128, n_features)
imp = saits.impute({"X": chunks})
filled = imp.reshape(512, n_features)
```

SAITS 实例通过 `_PRETRAINED_SAITS_CACHE` 缓存，避免每次重新加载。

### 4.4 Per-instance SAITS / BRITS（仅 sanity）

`methods/dynamics_impute.py` 内部还可以做 per-instance fit（SAITS / BRITS 在单条测试轨迹上从头训）。但**对 SAITS / BRITS 不公平**（数据不足以训练 transformer），所以只在附录 sanity check 用，不进主对照实验。

## 5 Forecaster 接口

### 5.1 统一签名

每个 forecaster 提供：
```python
forecast = forecaster_func(ctx_filled, pred_len)
# Input:  ctx_filled shape (T, D), no NaN
# Output: forecast shape (pred_len, D)
```

### 5.2 Panda-72M（headline forecaster）

`baselines/panda_adapter.py::panda_forecast` + `panda_model.py`

**架构**：12 层 PandaEncoder（temporal + channel self-attention）+ PandaHead（mean-pool over patches → linear projection）。**Encoder-only**，no decoder。

**调用**：
```python
from baselines.panda_adapter import panda_forecast
forecast = panda_forecast(filled, pred_len=128, device="cuda")
```

**注意**：Panda 内部把 `pred_len` round 到模型原生 `prediction_length`（典型 128）。如果传入 `pred_len < 128`，输出仍是 128-step；调用方需要 `forecast[:pred_len]` 截断。

**Patch tokenizer**：
- context 长度需要等于 `cfg.context_length` (= 512)；不足左 pad，超过取最后 512
- per-channel mean / std 归一化
- patch_length=16, patch_stride=16，所以 P = 32 patches per channel
- 输入到 encoder shape：`[B=1, C=D, P=32, patch_length=16]`

### 5.3 Chronos-bolt-small（cross-foundation forecaster）

`baselines/chronos_adapter.py::chronos_forecast`

**架构**：HuggingFace 上的 BoltChronosPipeline（univariate transformer）。

**调用**：
```python
from baselines.chronos_adapter import chronos_forecast
forecast = chronos_forecast(filled, pred_len=64, device="cuda")
```

**多变量处理**：Chronos 是 univariate，对 D 维输入逐通道独立预测，然后 stack：
```python
# pseudo
inputs = [torch.tensor(arr[:, d]) for d in range(D)]
out = pipe.predict(inputs, prediction_length=pred_len)  # 一次 batch 调用
# out.shape = (D, num_quantiles, pred_len) → median over quantile axis → (D, pred_len)
forecast = median.T  # (pred_len, D)
```

`num_samples=20` 默认（用于 quantile 估计）。

**Pred_len 警告**：Chronos library 警告 `prediction_length > 64` 超出训练分布。论文中 §6.4 同时报 `pred_len ∈ {64, 128}` 来排除"OOD horizon"是 negative result 的解释。

### 5.4 DeepEDM（延迟坐标 companion forecaster）

延迟坐标 forecaster（Takens 嵌入），使用 softmax-attention learned-kernel head [Majeedi25]。

每个系统训练独立的 DeepEDM。其 cell 参与 §3.2 与 Figure 2 的 imputer × forecaster 矩阵。

**关键事实**：DeepEDM 在 L96 N=20 v2 cross-system 上是**唯一**SP55–SP82 每个 cell 都拿到严格正 paired CSDI − linear CI 的 forecaster（§4.1）。

### 5.5 EnKF（已知动力学上界）

`experiments/week1/enkf_l63_upper_bound.py`

Stochastic ensemble Kalman filter：
- n_members = 100
- RK4 forward propagation 用真实 L63 向量场（与 forecast 时一致的动力学）
- Observation noise floor = 0.01 × σ_attr（数值稳定性）

只用作 model-aware reference，不参与 imputer × forecaster 矩阵。

## 6 评测度量

### 6.1 VPT — Valid Prediction Time（合成系统）

定义：forecast 与 ground-truth future 的相对 RMSE 首次超过阈值的时间，以 Lyapunov time 为单位。

```python
# from experiments/week1/lorenz63_utils.py
def valid_prediction_time(future_true, forecast, dt, lyap, threshold, attractor_std):
    err = forecast - future_true                         # (H, D)
    rmse = np.sqrt((err**2).mean(axis=1))                # per-step RMSE across features
    rel_rmse = rmse / attractor_std                      # 归一化到吸引子尺度
    above = rel_rmse > threshold
    if not above.any():
        return H * dt * lyap                             # 跑满了
    first_above = np.argmax(above)
    return first_above * dt * lyap                       # in Lyapunov units
```

**论文中报的 VPT 值**：
- `vpt03` / `vpt05` / `vpt10` 分别用 threshold = 0.3 / 0.5 / 1.0
- 主 headline 是 `vpt10`（VPT@1.0 Lyapunov time）

### 6.2 vh@τ — Normalized Valid Horizon（真实数据 §6.6）

真实数据（Jena）没有"clean trajectory"做 reference，所以用 z-RMSE 跨特征：

```python
# from experiments/week1/jena_real_sensor_pilot.py
def normalized_valid_horizon(future_true, forecast, threshold=1.0):
    err = future_true - forecast                  # both already z-scored
    per_step = np.sqrt((err * err).mean(axis=1))  # RMSE across z-features
    above = per_step > threshold
    return int(np.argmax(above)) if above.any() else H
```

返回 lead-step 数（不乘 Lyapunov）。论文报 vh@0.3 / vh@0.5 / vh@1.0 / vh@2.0。

### 6.3 Survival probability + Wilson CI

Tail metric：
- `Pr(VPT > 0.5 Λ)` — 弱 tail
- `Pr(VPT > 1.0 Λ)` — 强 tail（headline）

二项 CI 用 Wilson 95%（不是 normal approximation，n 小时更稳）：

```python
def wilson(k, n, z=1.96):
    if n == 0: return (0., 0.)
    p = k / n
    den = 1 + z**2/n
    centre = (p + z**2/(2*n)) / den
    half = z * sqrt(p*(1-p)/n + z**2/(4*n**2)) / den
    return centre - half, centre + half
```

报：`{N}/{N_total} = {%}, Wilson 95% [{lo%}, {hi%}]`

### 6.4 Paired bootstrap CI

Mean 差的 paired CI（5000 resamples）：

```python
def paired_bootstrap(a, b, n_boot=5000, seed=29):
    rng = np.random.default_rng(seed)
    diff = np.asarray(a) - np.asarray(b)        # paired per seed
    boots = np.array([
        diff[rng.integers(0, len(diff), len(diff))].mean()
        for _ in range(n_boot)
    ])
    return float(diff.mean()), float(np.quantile(boots, 0.025)), float(np.quantile(boots, 0.975))
```

**报的格式**：`Δ = +X.XX [+lo, +hi] ↑`（↑ = strict-positive，↓ = strict-negative，≈ = CI 跨 0）

### 6.5 度量约定

- **Mean** — bootstrap CI
- **Median + Pr(VPT > τ)** — L96 高方差 cell 的 headline（§4.3 预注册）
- **Direction-and-rank** — 当 mean 与 Wilson CI 都不严格分离但 per-seed rank 一致时使用

## 10 常见坑与排错

### 10.1 PYTHONPATH 不对

`ModuleNotFoundError: No module named 'experiments.week1'`
→ 用 `bash -c "cd /path/to/CSDI-PRO && PYTHONPATH=/path/to/CSDI-PRO python -u -m ..."`，**特别在 nohup 里**。

### 10.2 CSDI sigma_override 误传

**症状**：CSDI imputation 在观测时步严重偏离 ground-truth（anchor 误差 >> 10⁻⁶）。
**原因**：`sigma_override` 没传或传错（默认值与 cell 不匹配）。
**修法**：明确传 `sigma_override = noise_std_frac * attractor_std`，纯稀疏 cell（σ=0）传 0。

### 10.3 SAITS chunked inference 维度

**症状**：`size mismatch for encoder.embedding_1.position_enc.pos_table: copying a param with shape torch.Size([1, 128, ...]) from checkpoint, the shape in current model is torch.Size([1, 512, ...])`
**原因**：SAITS 训练时 `n_steps=128`，推理时传了 `n_steps=512`。
**修法**：`SAITS(n_steps=128, ...)` 然后把 context (512, D) 切成 (4, 128, D) 喂进去。

### 10.4 Panda 输出 pred_len 不对

**症状**：调用 `panda_forecast(filled, pred_len=64)` 但返回 shape (128, D)。
**原因**：Panda 内部 round 到 `cfg.prediction_length=128`。
**修法**：调用方显式 `forecast = panda_forecast(...)[:pred_len]` 截断。

### 10.5 Chronos pred_len 警告

**症状**：`UserWarning: prediction_length > 64 is outside the training distribution`
**原因**：Chronos-bolt-small 训练时 pred_len ≤ 64。
**处理**：论文中我们同时报 pred_len=64 与 128，确认 negative result 不是 OOD horizon 的 artefact。

### 10.6 L96 mean 高方差

**症状**：L96 SP65 / SP82 cell 上 Panda mean VPT std 比 mean 还大；paired CI 跨 0。
**原因**：少数 lucky linear seed 偶然落在好 token 区域，VPT 跑到 8–11，dominates mean。
**预注册修法**：L96 高维 cell 用 **median + survival** 作为 headline（§4.3），或者扩种子（如 L96 SP82 30 seeds, §4.4）。

### 10.7 CSDI sampler 跨实验数字微差

**症状**：L63 SP65 paired CSDI − linear 在 §3.2 是 +1.64，§4.3 是 +1.65，§4.4 是 +1.67。
**原因**：CSDI 是随机 diffusion sampler，每次独立 inference run 有 sampler stochasticity。
**处理**：在 Appendix B.3 显式声明这是 sampler stochasticity 不是协议漂移，CIs 互相重叠。
